### Day 12: Schedule Management (Mon, Nov 24)

**Topics Covered:**

**1. Schedule Management Plan** ðŸ”¥ðŸ”¥
- Guides team on schedule definition, approval, and control

**2. Define Activities**
- **Waterfall:** Activities defined for entire project upfront
- **Agile:** Activities defined per user story/iteration
- Continuous and iterative in Agile
- Creates Activity List

**3. Estimating Activity Durations** ðŸ”¥ðŸ”¥ðŸ”¥

**Waterfall Techniques:**

**A. Analogous Estimate (Top-Down)**
- Uses similar past projects
- Quick but less accurate
- Expert judgment based
- **Example:** Last website took 3 months, this one similar â†’ estimate 3 months

**B. Parametric Estimate**
- Uses statistical relationship
- Mathematical model
- More accurate than analogous
- **Formula:** Duration = Quantity Ã— Productivity Rate
- **Example:** 1000 lines of code Ã— 10 lines/hour = 100 hours

**C. Three-Point Estimate** ðŸ”¥ðŸ”¥ðŸ”¥
- Uses three scenarios
- Most common in PMP exam!
- **Formulas:**

**Triangular Distribution:**
```
Estimate = (Optimistic + Most Likely + Pessimistic) / 3
```

**PERT/Beta Distribution (More Weight on Most Likely):**
```
Estimate = (Optimistic + 4Ã—Most Likely + Pessimistic) / 6
```

**Example:**
- Optimistic (O) = 10 days
- Most Likely (M) = 15 days
- Pessimistic (P) = 26 days

Triangular: (10 + 15 + 26) / 3 = 17 days
PERT: (10 + 4Ã—15 + 26) / 6 = 16 days

**4. Agile Estimation Techniques** ðŸ”¥ðŸ”¥

**A. Planning Poker**
- Delphi method application
- Team members estimate independently
- Reveal estimates simultaneously
- Discuss differences
- Reach consensus
- **Cards:** Fibonacci sequence (1, 2, 3, 5, 8, 13, 21...)

**B. Affinity Estimation**
- Group similar items
- Relative sizing
- T-shirt sizes or story points
- Fast for large backlogs

**C. T-shirt Sizing**
- Categories: XS, S, M, L, XL, XXL
- Quick, relative estimation
- Good for early planning
- Later converted to story points

**5. Agile Schedule Metrics** ðŸ”¥ðŸ”¥

**A. Lead Time**
- Time from request to delivery
- Customer perspective
- **Example:** Request on Monday, delivered Friday = 5 days lead time

**B. Cycle Time**
- Time from start of work to completion
- Team perspective
- **Example:** Started Wednesday, done Friday = 2 days cycle time

**C. Response Time**
- Time from request to start of work
- **Example:** Request Monday, start Wednesday = 2 days response time
- **Formula:** Lead Time = Response Time + Cycle Time

**D. Work in Progress (WIP) Limit** ðŸ”¥
- Maximum items in progress simultaneously
- Prevents overloading team
- Improves flow
- **Example:** WIP limit of 3 = max 3 tasks in "In Progress" column

**Important:** Agile focuses on flow-based metrics (lead time, cycle time, WIP) instead of burndown charts!
